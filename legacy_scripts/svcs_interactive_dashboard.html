About Database Pruning

Pruning removes semantic data for commits that no longer exist in your git history. This can happen when:

  • Commits are rebased, squashed, or force-pushed
  • Branches are deleted
  • History is rewritten with git filter-branch

Warning: This operation permanently removes orphaned semantic data.

Recommendation: Make sure you have a backup of your database before running this operation.
